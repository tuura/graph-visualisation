      The Settings type used by most drawing modules to customise the drawing of a graph.

      data Settings Source #

      The Settings data type contains various fields for the graph drawing methods, some common between them and some specialised to a single method. To give custom settings to the drawGraph' function, a function which takes a graph as an argument and returns a Settings instance will be given to it.

      Constructors

      Settings 

      Fields

      • dynamicHead :: Measure Double

        The size of the arrow heads for the graph drawing, useful to scale with graph size. The default implementation for each of the drawing methods is similar, being based on: dynamicStyle small $ count g With normal being used in place of small for some methods.

      • dynamicThick :: Measure Double

        The thickness of the arrow shafts for the graph drawing, again useful to scale like dynamicHead like so: dynamicStyle thin $ count g

      • directed :: Directed

        Whether the graph is Directed (arrows on the connections between nodes). The type Directed is a boolean value that can either be Directed or Undirected.

      • horizontalOrientation :: Maybe Bool

        The orientation of the graph when using the Visualise.Tree module's functions.

      • layerSpacing :: Maybe Double

        The spacing between layers when using the Visualise.Tree module's functions.

      • nodeSpacing :: Maybe Double

        The spacing between vertices when using the Visualise.Tree module's functions.

      • graphPadding :: Maybe Double

        The padding around the graph when using the Visualise.Tree module's functions.

      • colF :: Maybe (Int -> Colour Double)

        The background colour of groups when using the Visualise.Hierarchical module.

      • bgOp :: Maybe Double

        The background opacity of groups when using the Visualise.Hierarchical module.

      • initPos :: Maybe Int

        The mode of initial positioning for the vertices when using Visualise.FlatAdaptive, changing this results in differently layed out graphs. Useful to experiment with to best display a graph.

      The default vertex-drawing functions.

      drawDefaultNode :: Show a => a -> Diagram B Source #

      Produces a default node á la drawDefaultNodeWithSize with a size of '0.1'.

      drawDefaultNodeWithSize :: Show a => Double -> a -> Diagram B Source #

      The default drawing function to produce a single node but of the specified size, in the form of a 'Diagram B', containing some showable text. Also makes the node a link, so if the drawn graph is saved as an SVG file each vertex is clickable with a JavaScript alert giving the vertex's name.

      drawDefaultEmptyNode :: Double -> Diagram B Source #

      The default circle Diagram for an empty node of the size provided.

      drawNodeWithEmptyFlag :: Show a => (a, Bool) -> Diagram B Source #

      The same as drawDefaultNode but instead takes a tuple of a showable value and a boolean flag. If the flag is True then it means the vertex should be displayed as an empty vertex with no contents, but if the flag is true the vertex is drawn as normal.

      dynamicStyle :: Measure Double -> Int -> Measure Double Source #

      Used to produce a measurement for the size of Diagram properties from a default measurement and the size of a graph as an Integer. Multiplies the default value by 10 and divides this new value by the size of the graph.
